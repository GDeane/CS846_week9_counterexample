def add_days(date_str: str, days: int) -> str:
    """
    Input:  date_str in 'YYYY-MM-DD'
    Output: date_str in 'YYYY-MM-DD' after adding days
    """
    try:
        dt = datetime.strptime(date_str, "%Y-%m-%d")

#### Guideline 3 & 6 (Revised): Enforce runtime inspection first + list edge cases explicitly at the end

#### Description

Before changing logic, require temporary debug outputs to reveal runtime state and where the exception occurs.

---

#### Prompt and Context

**Improved Prompt:**

date_utils.py crashes when running test_date_utils.py.

Before modifying any logic, do the following:

1. Add temporary debug prints inside add_days() to print:
   - the incoming date_str
   - the incoming days value
   - the exact line where the exception occurs

2. Tell me what command to run.

3. After I provide the debug output, suggest the smallest safe fix.

Important:

- Do not change the overall structure.
- Preserve behavior for valid dates.
- Handle invalid format.
- Handle impossible dates.
- Handle empty strings.

Keep edge cases in mind.

---

#### Solution or Output Generated by GitHub Copilot (GPT-4.1)

```python
from datetime import datetime, timedelta

def add_days(date_str: str, days: int) -> str:
    """
    Input:  date_str in 'YYYY-MM-DD'
    Output: date_str in 'YYYY-MM-DD' after adding days
    """
    print(f"[DEBUG] Incoming date_str: {date_str}")
    print(f"[DEBUG] Incoming days: {days}")
    try:
        dt = datetime.strptime(date_str, "%Y-%m-%d")
    except ValueError as ve:
        print("[DEBUG] Exception at datetime.strptime")
        raise ValueError(f"Invalid date format or value: '{date_str}'. Expected 'YYYY-MM-DD'.") from ve
    try:
        result_date = dt + timedelta(days=days)
    except Exception as e:
        print("[DEBUG] Exception at dt + timedelta")
        raise ValueError(f"Error adding days: {e}") from e
    return result_date.strftime("%Y-%m-%d")
```

#### Result

The improved prompt produced a better debugging outcome because:
- It forced runtime inspection first (Guideline 3 actually applied).
- It placed constraints and explicit edge cases at the end (Guideline 6 used correctly).
- It reduced “random fixes” and made the next step clear: run tests, read logs, then apply the smallest safe patch.
